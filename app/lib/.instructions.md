# Hướng dẫn viết code trong `app/lib/`

## `calculations.ts` — Tính công

Đây là file duy nhất chứa toàn bộ business logic tính ngày công. Không được đặt logic tính toán ở bất kỳ nơi nào khác.

### Các hàm cần có

```ts
/** Chuyển "HH:mm" thành số phút tính từ 00:00 */
function timeToMinutes(time: string): number;

/** Tính số phút làm thực tế trong ngày, đã khấu trừ nghỉ trưa */
function calcActualWorkMinutes(
	checkInTime: string,
	checkOutTime: string,
	settings: OrganizationSettings,
): number;

/** Tính hệ số công (workCoefficient) của một ngày */
function calcWorkCoefficient(
	actualMinutes: number,
	scheduledMinutes: number,
): number;

/** Phân loại trạng thái ngày công */
function classifyWorkDayStatus(
	log: WorkLog,
	settings: OrganizationSettings,
	isHoliday: boolean,
): WorkDayStatus;

/** Tổng hợp một ngày làm việc thành WorkDaySummary */
function summarizeWorkDay(
	log: WorkLog,
	settings: OrganizationSettings,
	isHoliday?: boolean,
): WorkDaySummary;

/** Tính tổng ngày công trong tháng cho một nhân viên */
function calcMonthlyWorkDays(summaries: WorkDaySummary[]): number;

/** Đếm số ngày làm chuẩn trong tháng (trừ cuối tuần và lễ) */
function countStandardWorkDays(
	year: number,
	month: number,
	settings: OrganizationSettings,
	holidays: string[],
): number;
```

### Quy tắc tính giờ

```
effectiveCheckIn  = max(actualCheckIn, workStartTime)
effectiveCheckOut = min(actualCheckOut, workEndTime)
rawWorkMinutes    = effectiveCheckOut - effectiveCheckIn

// Khấu trừ nghỉ trưa nếu khoảng làm việc giao với khung nghỉ trưa
lunchDeduction = overlap(
  [effectiveCheckIn, effectiveCheckOut],
  [lunchBreakStart, lunchBreakEnd]
)

actualWorkMinutes = rawWorkMinutes - lunchDeduction
workCoefficient   = actualWorkMinutes / scheduledMinutes
```

### Phân loại WorkDayStatus — thứ tự ưu tiên

1. `weekend` — ngày không thuộc `settings.workDays`
2. `holiday` — nằm trong danh sách lễ
3. `absent` — ngày làm việc nhưng không có check-in/check-out
4. `overtime` — `checkOutTime > workEndTime` và `workCoefficient > 1.0`
5. `late` — `checkIn > workStartTime + allowLateMinutes`
6. `early-leave` — `checkOut < workEndTime - allowEarlyLeaveMinutes`
7. `half-day` — `workCoefficient < 0.75`
8. `present` — mặc định

## `validators.ts` — Zod schemas

Validate **tại system boundaries**: dữ liệu form input và API response.

```ts
const organizationSettingsSchema = z.object({
	workStartTime: z.string().regex(/^\d{2}:\d{2}$/),
	workEndTime: z.string().regex(/^\d{2}:\d{2}$/),
	lunchBreakStart: z.string().regex(/^\d{2}:\d{2}$/),
	lunchBreakEnd: z.string().regex(/^\d{2}:\d{2}$/),
	standardWorkHoursPerDay: z.number().min(1).max(24),
	workDays: z.array(z.number().int().min(1).max(7)).min(1),
	timezone: z.string(),
	allowLateMinutes: z.number().int().min(0),
	allowEarlyLeaveMinutes: z.number().int().min(0),
});

const workLogFormSchema = z.object({
	employeeId: z.string().min(1, 'Vui lòng chọn nhân viên'),
	date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, 'Ngày không hợp lệ'),
	checkInTime: z
		.string()
		.regex(/^\d{2}:\d{2}$/)
		.nullable(),
	checkOutTime: z
		.string()
		.regex(/^\d{2}:\d{2}$/)
		.nullable(),
	notes: z.string().max(500),
});
```
