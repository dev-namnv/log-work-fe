# Hướng dẫn viết Services trong `app/apis/`

## Cấu trúc thư mục

```
app/apis/
  http.ts              # HTTP client lõi — không sửa trực tiếp
  auth.service.ts      # AuthService
  organization.service.ts
  work-log.service.ts
  notice.service.ts
  account.service.ts   # Admin only
  API.md               # Tài liệu API gốc (tham khảo)
```

## Quy tắc bắt buộc

1. **Tất cả service viết dạng `class` với method `static`** — không dùng singleton object hay export function rời.
2. Không `throw` trực tiếp trong service — để `http` client xử lý và ném `ApiException`.
3. Mỗi method tương ứng với đúng một API endpoint.
4. DTOs (Request/Response) khai báo bằng `interface` trong cùng file service.
5. Types domain (`Account`, `Organization`, `WorkLog`…) import từ `~/types/api`.

## `http.ts` — Client lõi

```ts
import { http } from '~/apis/http';

// Các method: get, post, put, patch, delete
// Tất cả tự động gửi cookie (credentials: 'include')
// Throw ApiException nếu response không ok
```

### Config endpoint

Biến môi trường: `API_BASE_URL` (server) hoặc `VITE_API_BASE_URL` (client).
Mặc định: `http://localhost:8080`.

## Mẫu viết service

```ts
import { http } from '~/apis/http';
import type { SomeDomainType } from '~/types/api';

// --- DTOs ---
export interface CreateSomethingDto {
	name: string;
}

export interface MessageResponse {
	message: string;
}

// --- Service ---
export class SomeService {
	static getAll(): Promise<SomeDomainType[]> {
		return http.get('/some-resource');
	}

	static getById(id: string): Promise<SomeDomainType> {
		return http.get(`/some-resource/${id}/detail`);
	}

	static create(dto: CreateSomethingDto): Promise<SomeDomainType> {
		return http.post('/some-resource', { json: dto });
	}

	static update(
		id: string,
		dto: Partial<CreateSomethingDto>,
	): Promise<SomeDomainType> {
		return http.patch(`/some-resource/${id}`, { json: dto });
	}

	static delete(id: string): Promise<MessageResponse> {
		return http.delete(`/some-resource/${id}/delete`);
	}
}
```

## Sử dụng trong Route loader/action

```ts
// Trong loader — server-side, request object có cookie
export async function loader({ request }: Route.LoaderArgs) {
	try {
		const profile = await AuthService.getProfile();
		return { profile };
	} catch (err) {
		if (err instanceof ApiException && err.statusCode === 401) {
			throw redirect('/login');
		}
		throw err;
	}
}

// Trong action
export async function action({ request }: Route.ActionArgs) {
	const formData = await request.formData();
	const dto = loginSchema.parse(Object.fromEntries(formData));
	const result = await AuthService.login(dto);
	return redirect('/');
}
```

## Pagination / Search

Các endpoint `/search` nhận body và trả về `PaginatedResponse<T>`:

```ts
import type { PaginatedResponse } from '~/types/api';

static search(dto: SearchDto): Promise<PaginatedResponse<SomeThing>> {
  return http.post('/some-resource/search', { json: dto });
}
```
